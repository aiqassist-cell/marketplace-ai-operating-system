# Спецификация: Первая волна агентов (Маркетолог, Data Collector, SMM, Контент-дизайнер)

Дата: 2026-07-07
Статус: согласовано с владельцем, готово к реализации

## Контекст

Дополняет `agent-creation-order.md` и `virtual-office-technical-architecture.md`. Здесь фиксируется операционная механика для конкретной первой волны из 4 агентов, которую владелец решил запускать вместе, а не строго по исходной очерёдности.

## Состав волны

1. **Marketer Strategist** — уже описан в `agents/marketer-strategist.md`
2. **Data Collector** — уже описан в `agents/data-collector.md`, здесь уточняется область сбора
3. **SMM / Distribution / World Manager** — уже описан в `agents/smm-world-manager.md`
4. **AI Production Lead** (`agents/ai-production-lead.md`, AGENT-007, уже существует) — производство визуального контента для соцсетей (карусели, слайды-подборки, UGC-видео) через Higgsfield. Отдельный новый агент не создаётся — эта роль уже покрывает генерацию по брифам, в неё встраивается Higgsfield как конкретный инструмент.
5. **Digital Stylist** (`agents/digital-stylist.md`) — **добавлен в волну решением владельца 2026-07-10** после инцидента с артикулом 237064803 (стилистический брак: тёплый кардиган на летнем платье, аксессуары без референсов, чужой цветовой вариант). Его карта готового образа — обязательный вход AI Production: без утверждённой владельцем карты образа генерация не запускается. Рич-контент карточки WB по-прежнему делает человек — в периметре агента только образы/стилизация для соцконтента.

Разделение SMM / AI Production Lead: SMM решает ЧТО и КОГДА публиковать (контент-план, хуки, календарь, тренды); AI Production Lead ПРОИЗВОДИТ сам визуальный актив по брифу SMM через Higgsfield.

## Вне периметра (явно)

- Карточки Wildberries и рич-контент к ним — делает человек-дизайнер, агенты не трогают.
- n8n — не подключается на этом этапе. Роль оркестратора временно у skill `schedule` (облачные routines).

## Запуск и расписание

- Механизм: skill `schedule` (облачные routines, персистентные — не завязаны на открытую сессию чата и не истекают через несколько дней, в отличие от сессионного инструмента `CronCreate`). Не n8n. n8n подключается позже как оркестратор поверх тех же самых инструментов (Apify, Higgsfield и т.д.) — они не заменяются, а вызываются n8n по цепочке.
- Каждый агент работает по своему расписанию (частота определяется в его собственном файле агента).

## Точки подтверждения человеком

| Решение | Режим |
|---|---|
| Цены / позиционирование (Marketer) | Всегда требует подтверждения владельца |
| Карта готового образа (Digital Stylist) | Утверждается владельцем ДО генерации — кредиты не тратятся на неутверждённые луки (решение 2026-07-10) |
| Публикация в соцсети | Требует подтверждения в пилотный период (см. ниже) |
| Трата кредитов Higgsfield (генерация) | Требует подтверждения в пилотный период + только по утверждённой карте образа |
| Сбор/мониторинг данных (Data Collector) | Автономно, без подтверждения — это наблюдение, не действие вовне |

### Пилотный период и переход к автономии

- Порог: **10 подряд одобренных публикаций** на площадке.
- По достижении порога агент **не переключается сам**. Он пишет в Airtable (таблица Tasks) статус вида `Готов к автономии — подтвердите` и ждёт явного решения владельца: продолжаем тест / переходим на автономную публикацию.
- Это та же механика, что и обработка ошибок (см. ниже) — агент останавливается на развилке и ждёт, а не гадает.

## Data Collector — область сбора

Сбор ведётся **широко с первого дня**, без ограничения по площадкам публикации — это наблюдение, риска и затрат почти нет (в отличие от публикации и генерации).

Источники: Wildberries (цены конкурентов, отзывы, остатки, бестселлеры, трендовые поисковые запросы) + Instagram, TikTok, Pinterest, YouTube, Telegram.

Инструмент: **Apify** (платно, usage-based, есть free tier) — покрывает WB и все 5 площадок готовыми акторами. Аккаунт заводится сейчас. При подключении n8n Apify остаётся тем же исполнителем скрейпинга — n8n только вызывает его по расписанию, не заменяет.

### Два типа трендов (важное отличие от изначального плана)

1. **Контент-тренды** — форматы, хуки, что залетает в ленте → адресат: SMM, AI Production Lead
2. **Товарные/фэшн-тренды** — силуэты, цвета, стили → адресат: Marketer, Productologist (существующий агент)

Лучшие источники товарных трендов: Pinterest (эстетика/стиль), сам Wildberries (бестселлеры, поисковые тренды — самый прямой сигнал для WB-only бренда), Instagram (фэшн-блогеры/хэштеги). TikTok/YouTube — больше про контент-тренды, чем товарные.

Новое поле в Airtable создавать не нужно: в таблице `Hypotheses` уже есть поле `Type` со значениями `Demand` (→ товарные тренды) и `Content` (→ контент-тренды) — Data Collector использует существующую классификацию.

## Публикация — пилот

- Публикуем нативными API платформ (Meta Graph API, TikTok Content Posting API, Pinterest API, YouTube Data API, Telegram Bot API), **не** через сторонний планировщик (Buffer/Later и т.п.).
- **Пилотные площадки: Telegram + Pinterest.** Причины: (а) минимальное ревью приложения — Telegram Bot API и Pinterest API не требуют долгой проверки, в отличие от Meta/TikTok (может занимать от нескольких дней до недель); (б) Pinterest хорошо соответствует ЦА и позиционированию QWEAR (женщины 35-43, quiet luxury, эстетика подборок/мудбордов).
- Регистрация приложений на остальных площадках (Meta, TikTok, YouTube) идёт **параллельно**, в порядке очереди, не блокируя разработку агентов — но публикация туда открывается только после того, как пилот на Telegram+Pinterest подтвердит рабочую механику.
- Разные площадки — разные форматы (что залетает в Pinterest ≠ что залетает в Telegram). Контент-бриф для AI Production Lead включает поле "площадка" с её форматными требованиями (поле `Platform` уже есть в `Content Assets`); используются и расширяются существующие шаблоны `templates/winner-card.md` и `templates/photo-ab-test.md`. Перед публикацией контент прогоняется через `virality_predictor` (Higgsfield MCP) для оценки хука/удержания под конкретный формат.

## AI Production Lead — инструмент генерации

Higgsfield подключён как **MCP-коннектор** (не CLI-skill — тот был удалён из-за неограниченного Bash-права и заменён на MCP как более контролируемый вариант). Аккаунт активен: план Plus, 110 кредитов на момент написания.

Основные вызовы: `generate_image` (карусели/слайды), `generate_video` / Shorts Studio / Marketing Studio (UGC-видео), `virality_predictor` (оценка перед публикацией). Встраивается в раздел "Prompt Pack" файла `agents/ai-production-lead.md` как дефолтный инструмент под соцконтент.

## Связь между агентами

Основной механизм: **статусные поля в Airtable** (Products, Tasks, Content Assets, Hypotheses, Publications, Results — схема уже намечена в `airtable-mvp-design.md`, новых полей заводить не нужно: `Hypotheses.Type` и `Content Assets.Platform` уже покрывают классификацию трендов и площадок). Агенты по своему расписанию проверяют таблицы на новые записи со статусом "готово к обработке" — без прямых вызовов друг друга.

**Одно исключение**: Data Collector → SMM. После сбора свежих контент-трендов Data Collector сразу триггерит SMM, не дожидаясь следующего слота по расписанию — тренды в соцсетях устаревают быстро, ждать нельзя.

## Обработка ошибок

Без автоповторов. Если задача агента не выполнилась (Apify не ответил, Higgsfield вернул ошибку, кончились кредиты) — агент пишет статус ошибки в Airtable (таблица Tasks) и останавливается, ожидая владельца. Тот же паттерн, что и для развилки "продолжаем пилот / переходим на автономию".
