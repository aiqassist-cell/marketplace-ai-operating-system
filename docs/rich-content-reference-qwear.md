## Главный вывод

Текущий rich-контент QWEAR продает не характеристики платья, а сценарии стилизации:

```text
одна базовая модель -> много образов -> простые правила -> ожидание результата -> покупка
```

Это хороший формат для Digital Stylist / Content Funnel Designer.

## Что Работает

### 1. Большое обещание

Пример:

```text
ОДНО ПЛАТЬЕ - ДЕСЯТКИ ОБРАЗОВ
```

Смысл: покупатель видит не одну вещь, а базу гардероба.

### 2. Простая польза

Примеры:

```text
Легко стилизовать
Комфортно носить
Работает в десятках образов
```

Смысл: платье не требует сложных решений, но выглядит вариативно.

### 3. Совет стилиста

Паттерн:

```text
СОВЕТ СТИЛИСТА:
одно действие -> другой силуэт / настроение / сценарий
```

Примеры:

- смените обувь;
- добавьте ремень;
- добавьте жакет;
- носите с джемпером или свитшотом;
- добавьте кеды, шоппер и очки.

### 4. До / после или вариантность

Показывать:

- день / вечер;
- без ремня / с ремнем;
- casual / smart casual;
- город / отдых;
- один товар / несколько настроений.

### 5. Сохранение как CTA

Пример:

```text
Сохраняйте в гардероб базу, которую будете носить снова и снова
```

Это полезно для соцсетей и карточки: CTA не давит покупкой, а просит сохранить решение.

## Повторяемые Блоки Rich-Контента

### Блок 1. Главное обещание

```text
Одно платье - десятки образов
```

### Блок 2. Галерея вариантов

5-7 образов на одной модели:

- повседневный;
- прогулка;
- отпуск;
- кафе / город;
- smart casual;
- вечер;
- прохладная погода.

### Блок 3. День / вечер

```text
От дня к вечеру:
смените обувь, сумку и украшения
```

### Блок 4. Многослойность

```text
Носите с джемпером, свитшотом, жакетом или рубашкой
```

### Блок 5. Силуэт

```text
Хотите другой силуэт? Добавьте ремень
```

### Блок 6. Городской образ

```text
Кеды + шоппер + очки
```

### Блок 7. Smart casual

```text
Добавьте структурный жакет и светлую обувь
```

## Формула Слайда

```text
Крупный заголовок:
Короткое действие:
Совет стилиста:
Визуальное доказательство:
Мягкий CTA:
```

## Требования К Digital Stylist

Агент должен уметь на основе одной модели создавать:

- 5-10 сценариев носки;
- наборы аксессуаров;
- идеи слоев;
- день/вечер;
- casual/smart casual;
- варианты с ремнем и без;
- сценарии для прохладной погоды;
- блоки для карточки;
- сценарии для TikTok/Reels/Shorts;
- rich-контент с CTA на сохранение.

## Как Перевести В Контентную Воронку

```text
Касание 1 - внимание:
Одно платье - десятки образов

Касание 2 - польза:
Платье не требует сложных решений

Касание 3 - доказательство:
5 вариантов стилизации

Касание 4 - возражение:
Слишком просто? Добавьте ремень / жакет / другой слой

Касание 5 - переход:
Сохраните базу в гардероб / смотрите карточку
```

## Что Улучшить В Следующих Версиях

- добавить блок "кому подойдет";
- добавить блок по посадке и длине;
- добавить блок "на какой рост / тип фигуры";
- добавить микродоказательства ткани: не просвечивает, держит форму, комфортно садится;
- добавить связь с размерной сеткой;
- добавить отдельные версии для plus size, если модель подходит;
- добавить более явные CTA для карточки маркетплейса.
