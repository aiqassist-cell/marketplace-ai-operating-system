# Seller AI Team

Операционная система цифровых сотрудников для небольшого бренда-селлера маркетплейса.

Цель репозитория: хранить роли, процессы, шаблоны, контентную систему, аналитику и правила работы AI-команды так, чтобы не смешивать стратегию, производство контента и ежедневную операционку.

## Главная логика

```text
Товар -> смысл -> мир / готовое решение -> контент -> дистрибуция -> метрики -> winners -> масштабирование -> продажи
```

## Базовые блоки

- `agents/` - цифровые сотрудники и зоны ответственности.
- `content-factory/` - креативная база, production blueprint, дистрибуция, winners и цикл оптимизации.
- `products/` - ассортимент, конкуренты, отзывы, готовые решения.
- `brand/` - позиционирование, ЦА, офферы, tone of voice.
- `templates/` - шаблоны брифов, отчетов, гипотез и контент-плана.
- `docs/` - архитектура системы и принятые решения.
- `pilots/` - тестовые прогоны агентской цепочки на реальных товарах.

## Принципы

1. СММ не дублирует маркетолога. Маркетинг отвечает за "что, кому и почему продаем", СММ отвечает за "как это заметят, досмотрят, сохранят и перейдут".
2. Контент-завод для небольшого бренда запускается как контент-лаборатория: маленькие циклы, четкие гипотезы, быстрые выводы.
3. AI-блогер работает только если у него есть мир: среда, конфликт, антагонисты, правила, рубрики и повторяемые сценарии.
4. Digital Stylist продает не товар, а готовый результат: образ, систему применения, пользу и ожидание результата.
5. Масштабируем только winners, слабое заменяем быстро.
